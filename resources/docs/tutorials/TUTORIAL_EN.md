## 1. Getting Started

### 1.1 Installation

```bash
conda env create -f environment.yml
conda activate alas
python main.py
```

### 1.2 Login / Register

On launch, ALAS shows a login screen. Create an account or sign in with existing credentials. Your session is stored locally so you won't need to log in every time.

## 2. Loading Data

ALAS works with `.las` and `.laz` point cloud files.

### 2.1 Single file

**File → Open** (`Ctrl+O`) — opens a file picker. After loading, the cloud appears in the 3D viewport and a new layer is added to the Layers panel.

### 2.2 Multiple files / tiles

**File → Open Multiple** (`Ctrl+Shift+O`) — select several files at once. Each file becomes its own layer. Use **Process → Merge Tiles** afterwards to combine them into a single cloud.

### 2.3 Large files

Files with more than ~1 million points are automatically decimated on import to keep the viewport responsive. The original data is preserved on disk; the decimated version is only used for display and interactive analysis.

### 2.4 Coordinate Reference System

ALAS reads the CRS from the LAS/LAZ header. If none is found, a dialog will ask you to enter an EPSG code (e.g. `25830` for ETRS89 UTM zone 30N Spain).

## 3. 3D Viewport

The central panel is an interactive 3D view powered by PyVista/VTK.

### 3.1 Navigation

| Action | Control |
|--------|---------|
| Rotate | Left-click drag |
| Pan | Middle-click drag |
| Zoom | Scroll wheel |
| Reset view | `R` |

### 3.2 Preset views

| View | Shortcut |
|------|---------|
| Top (plan) | `T` |
| Front | `F` |
| Side | `S` |

### 3.3 Colorization modes

Open the **Properties** panel (right side) to change how points are colored:

- **Height (Z)** — elevation gradient, useful for terrain overview
- **Intensity** — LiDAR return intensity, highlights roofs and road surfaces
- **Classification** — color per class code (ground, vegetation, building…)
- **Return Number** — first / last returns
- **Original RGB** — true color when the file carries RGB channels
- **Solid color** — single uniform color with adjustable point size

### 3.4 Point size

Drag the **Point Size** slider in the Properties panel to increase or decrease point rendering size.

## 5. Classification

**Process → Classify** opens the Classification dialog.

### 5.1 Available algorithms

| Algorithm | Best for |
|-----------|---------|
| **SMRF** (Simple Morphological Filter) | Open terrain, sparse vegetation |
| **CSF** (Cloth Simulation Filter) | Steep slopes, complex terrain |
| **PMF** (Progressive Morphological Filter) | Urban areas |
| **AI** (Neural Network) | Mixed scenes — requires a `.pt` model file placed in `resources/models/` |

### 5.2 Options

- **Post-process: vegetation** — classifies low / medium / high vegetation after ground separation
- **Post-process: buildings** — attempts building roof extraction

### 5.3 Workflow

1. Select the target layer in the Layers panel.
2. Open **Process → Classify**.
3. Choose the algorithm and tune parameters.
4. Click **Run**. A progress bar appears; the viewport updates when finished.
5. Switch the colorization to **Classification** to inspect results.

### 5.4 Classification history

**Process → Classification History** (`Ctrl+Shift+H`) shows a log of all runs with parameters and per-class point counts. Useful for comparing algorithm settings.

## 6. Digital Models (DTM / DSM / CHM)

**Process → Generate DEM** opens the raster generation dialog.

| Model | Description |
|-------|-------------|
| **DTM** | Digital Terrain Model — ground points only |
| **DSM** | Digital Surface Model — highest return per cell |
| **CHM** | Canopy Height Model — DSM minus DTM |

### 6.1 Interpolation methods

- **IDW** — Inverse Distance Weighting, smooth results, good for sparse data
- **TIN** — Triangulated Irregular Network (Delaunay), preserves break-lines
- **Nearest Neighbor** — fastest, good for dense uniform clouds

### 6.2 Steps

1. Ensure the point cloud is classified (ground class must be present for DTM/CHM).
2. Open **Process → Generate DEM**.
3. Select the model type, cell resolution (meters), and interpolation method.
4. Click **Generate**. The raster is added as a new layer and auto-exported as GeoTIFF.

## 7. Analysis Modules

All modules are under the **Analysis** menu. Each requires at least one layer to be selected.

### 7.1 Geomorphology

**Analysis → Geomorphology** computes terrain derivatives from a DTM:

- **Slope** — gradient in degrees
- **Aspect** — cardinal direction of slope face
- **Curvature** — profile and plan curvature
- **Roughness** — local surface variability
- **Hillshade** — shaded relief for visualization
- **Morphometric classification** — landform classes (ridges, valleys, flats…)

Select the desired outputs, set the DTM layer, and click **Run**. Each output becomes a separate raster layer.

### 7.2 Hydrology

**Analysis → Hydrology** models water flow over the terrain:

- **Flow Direction (D8)** — single-direction flow routing
- **Flow Accumulation** — upstream contributing area per cell
- **Ponding Zones** — depressions that trap water
- **Rainfall Simulation** — simulates a rainfall event and traces runoff
- **Flood Simulation** — inundation at a given water level

Set the DTM layer and the desired outputs, then click **Run**.

### 7.3 Vegetation

**Analysis → Vegetation** works on classified point clouds with vegetation classes:

- **Tree Detection** — identifies individual tree locations
- **Crown Segmentation** — delineates individual tree crowns
- **Density Map** — raster of point density per cell

Results are exported as raster layers and optionally as vector files.

### 7.4 Multitemporal Analysis

**Analysis → Multitemporal** compares two DTMs from different dates:

- **DEM of Difference (DoD)** — cell-by-cell elevation change
- **Change Classification** — erosion / deposition / stable zones
- **Deforestation Detection** — canopy loss between epochs

Steps:
1. Load DTMs from two different dates (or generate them from classified clouds).
2. Open **Analysis → Multitemporal**.
3. Assign the *before* and *after* DTM layers.
4. Select outputs and click **Run**.

### 7.5 My Reports

**Analysis → My Reports** (`Ctrl+Shift+R`) lists all PDF reports generated in previous sessions. Click any entry to open or re-export it.

## 8. Measurement Tools

All tools are under the **Tools** menu. Activate a tool, interact with the viewport, and results appear in the **Measurements** panel.

### 8.1 Topographic Profile

**Tools → Profile** — click two points in the viewport to draw a cross-section. A chart shows elevation along the transect.

### 8.2 Distance

**Tools → Distance** — click two points to measure the straight-line 3D distance and its horizontal (2D) projection.

### 8.3 Area

**Tools → Area** — click vertices to define a polygon, then right-click to close it. Reports planimetric area (flat) and surface area (computed from the underlying DTM).

### 8.4 Volume

**Tools → Volume** — define a polygon as above, then set a reference elevation. Reports cut and fill volumes relative to that plane.

### 8.5 Measurements history

**Tools → Measurement History** (`Ctrl+H`) shows all measurements from the current session. Individual rows can be copied or exported to CSV.

Press `Esc` to cancel any active measurement tool.

## 9. Batch Processing

**Process → Batch** (`Ctrl+B`) lets you run the same pipeline on a folder of LAS/LAZ files without loading each one manually.

### 9.1 Workflow

1. Open **Process → Batch**.
2. Set the **Input folder** containing your `.las`/`.laz` files.
3. Set the **Output folder** for results.
4. Build the pipeline by enabling steps in order:
   - Preprocessing (noise filter, decimation)
   - Classification (choose algorithm)
   - DEM generation (choose model type and resolution)
   - Analysis (geomorphology, hydrology, vegetation)
   - Export (GeoTIFF, PDF report)
5. Click **Start**. A progress bar tracks file-by-file and step-by-step progress.

Batch runs can be paused and results are saved incrementally, so a crash does not restart the whole queue.

## 10. Export

**File → Export** (`Ctrl+E`) opens the Export dialog for the selected layer.

### 10.1 Point cloud formats

| Format | Notes |
|--------|-------|
| **LAZ** | Compressed LAS, recommended for archiving |
| **LAS** | Uncompressed, maximum compatibility |

### 10.2 Raster

- **GeoTIFF** — standard georeferenced raster, compatible with QGIS, ArcGIS, GDAL

### 10.3 3D Mesh

- **OBJ** — triangulated mesh derived from the point cloud, usable in Blender, MeshLab

### 10.4 Report

- **PDF** — summary report with statistics, colorized renders, and analysis results. Generated reports are saved in the cloud and accessible from **Analysis → My Reports**.

## 11. Projects

| Action | Menu / Shortcut |

| Save project | **File → Save Project** (`Ctrl+S`) |

Opening a `.alas` file restores all layers and their settings exactly as you left them.

## Tips

- **Workflow order:** Load → Preprocess → Classify → Generate DTM → Analyse. Following this order avoids missing inputs in downstream steps.
- **Large projects:** Use Batch Processing instead of loading all tiles interactively. It is significantly faster and uses less RAM.
- **CRS matters:** Always verify the EPSG code on import. Wrong CRS will misplace data and corrupt measurements.
- **AI Classification:** Place your `.pt` model file in `resources/models/` before opening the Classification dialog, otherwise the AI option will be greyed out.
- **PDF reports:** Reports are stored server-side. They are accessible from any machine after login via **Analysis → My Reports**.
